# Uptime Kuma

Uptime Kuma is a self-hosted monitoring tool like "Uptime Robot".

## Prometheus Metrics

Uptime Kuma v2 natively exposes Prometheus metrics at the `/metrics` endpoint. In this setup, Prometheus is configured to scrape these metrics using basic authentication with an API key as the password that's passed to prometheus via a Kubernetes secret.

### Monitor Metrics

These are the primary metrics generated by the actual checks you configure in the Uptime Kuma UI. They are highly useful for creating Grafana dashboards or triggering Prometheus alerts.

*   `monitor_status`: The current status of the monitor (1 = UP, 0 = DOWN, 2 = PENDING, 3 = MAINTENANCE).
*   `monitor_response_time`: The response time of the service in milliseconds.
*   `monitor_response_time_seconds`: The average response time over a sliding window.
*   `monitor_uptime_ratio`: The calculated uptime SLA percentage (0.0 to 1.0) over a sliding window.
*   `monitor_cert_days_remaining`: How many days remain until the SSL/TLS certificate expires.
*   `monitor_cert_is_valid`: Whether the certificate is currently valid (1 = Yes, 0 = No).

**Labels (Tags)**
When you create monitors, the metrics above will automatically be tagged with the following labels so you can filter and group them:
*   `monitor_name`: The friendly name you gave the monitor.
*   `monitor_type`: The type of check (e.g., `http`, `ping`, `tcp`, `keyword`).
*   `monitor_url` / `monitor_hostname`: The target being monitored (if applicable).

### Application / System Health Metrics

Because Uptime Kuma is built on Node.js and Express, it also exposes system health metrics to monitor the application itself:

*   **HTTP Metrics**: `http_request_duration_seconds`, `http_request_size_bytes`, `http_response_size_bytes`
*   **Express Server**: `expressjs_number_of_open_connections`
*   **Node.js Diagnostics**: `nodejs_eventloop_lag_seconds`, `nodejs_gc_duration_seconds`, `nodejs_active_handles`, `nodejs_active_requests`
*   **Resource Usage**: `process_resident_memory_bytes`, `process_cpu_seconds_total`, `nodejs_heap_size_used_bytes`, `process_open_fds`
*   **Version Info**: `app_version` (e.g., `2.1.3`)
